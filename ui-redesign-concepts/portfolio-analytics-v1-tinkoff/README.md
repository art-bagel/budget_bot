# Portfolio · Аналитика — концепт v1 (Tinkoff-leaning)

Редизайн UX **раздела «Аналитика» внутри страницы Портфель** в том же языке, что и `portfolio-v1-tinkoff` / `dashboard-v2-tinkoff`: серый фон, жёлтый акцент `#FFDD2D`, Onest, плотная информативная подача. Статический прототип без сборки.

## Главный сдвиг UX

Раньше «Аналитика» — одна общая карточка с 4 метриками, одинаковая независимо от активной вкладки активов. Теперь:

> **Аналитика подстраивается под активную вкладку.** Выбрали «Все» — общая аналитика по портфелю. Перешли на «Депозиты» — другие модули, другие цифры, другой бенчмарк.

Плюс **hero морфирует**, когда вы переходите во view `Аналитика`:

| | Позиции / Операции (старый hero) | Аналитика (новый hero) |
|---|---|---|
| **Eyebrow** | «Сейчас в портфеле» | «Доходность · 12 месяцев» |
| **Toggle** | «Сейчас / С доходом» | Период: `1М / 3М / 6М / 1Г / Всё` |
| **Big number** | Баланс `2 665 000 ₽` | Прирост `+218 600 ₽` |
| **Delta line** | `+1,8% за 30 дней` | `+8,9% · обогнал MOEX на +3,5 п.п.` |
| **Breakdown rows** | Сколько лежит по типам | Сколько принёс каждый тип за период (₽ + % бейдж) |

Один и тот же hero отвечает на разные вопросы в зависимости от того, в каком разделе пользователь находится. На таб-уровне breakdown-строки фильтруются: на `Депозиты` остаётся одна строка с депозитами.

## Как открыть

Двойной клик по `index.html`. В DevTools — Toggle device toolbar → iPhone 14 Pro / Pixel 7 (375–414 px).

## Модули аналитики (по приоритету сверху вниз)

### 0. Strip «scope» — что мы сейчас анализируем

Между переключателем view и первой карточкой — узкая полоска с иконкой типа актива, заголовком (например, «Ценные бумаги») и pill-значением доходности. Якорь, который не даёт потерять контекст при скролле.

### 1. Откуда пришёл доход (Source-of-return)

Горизонтальный stacked-bar из 4 сегментов: `Прирост капитала · Купоны · Дивиденды · Проценты по вкладам`. Под баром — легенда: точка + название + ₽ + % от общего.

Принципиально: разные асс-вкладки показывают разный состав:

- **Все** — все 4 сегмента
- **Бумаги** — прирост, купоны, дивиденды (без процентов по вкладам)
- **Депозиты** — только проценты (1 сегмент = 100%)
- **Крипта** — только прирост капитала
- **Прочее** — только прирост капитала / доходность LQDT

Когда у класса один источник, бар «свёрнут» в одну заливку, и легенда показывает только применимое — пользователь сразу видит «инструмент даёт доход только одним способом».

### 2. По месяцам

12 вертикальных баров, центрированных на нулевой линии. Зелёные вверх (прибыль), красные вниз (убыток). Под каждым баром — буква месяца. Клик по бару заполняет блок ниже («Апрель 2026 · +44 600 ₽ · лучший месяц года»).

Альтернатива calendar-heatmap — выбраны вертикальные бары: они дают честный визуальный масштаб разницы, тогда как тепловая карта стирает её цветовой шкалой.

### 3. Лидеры и аутсайдеры (center-zero bars)

Список позиций, отсортированных по вкладу в доход за период. Бар идёт от центра: вправо в зелёном (победители), влево в красном (аутсайдеры). Лидер #1 получает жёлтый ранг-чип — единственное место на странице, где жёлтый используется как «корона».

На вкладке `Депозиты` модуль не показывается (все вклады плюсуют примерно одинаково — лучше отдать место «Прогресс вкладов»).

### 4. Распределение — donut (только `Все`)

170 × 170 SVG donut с 4 сегментами в палитре `ink/mint/coral/grape`. В центре — крупный процент крупнейшей доли («Бумаги 49,2%»). Под донатом — двухколоночный legend с %% и ₽.

### 5. Прогресс по вкладам (только `Депозиты`)

Список вкладов с прогрессбарами **срока**. Жёлтая заливка + чёрный «маркер» в текущей точке. Под прогрессбаром — `% срока · до 14 июля 2026` и `+32 400 ₽ накоплено`. Бессрочные вклады («до востребования») получают полосатую штриховку вместо градиента — сразу видно, что это другой тип.

Внизу 3 KPI: средневзвешенная ставка, накоплено за год, дней до ближайшей выплаты.

### 6. Дистанция до пика (только `Крипта`)

Для каждой монеты — горизонтальная «температура» от 0 до ATH с курсором в текущей позиции и жёлтой меткой пика справа. Под полоской — текущая цена и ATH с датой. Цвет градиента полоски смещён в коралл/красный — намёк, что в крипте «полная заливка» обычно означает «надо фиксироваться».

### 7. Активность сделок (только `Бумаги`)

3 KPI: количество сделок (с разбивкой на покупки / продажи), оборот, нетто. Ниже — 2 строки полученных выплат (дивиденды + купоны) с тонкими столбиками.

### 8. Сравнение с бенчмарком (все вкладки, но контент разный)

Горизонтальные бары с эталонами. На `Все`: `Портфель / MOEX TR / Инфляция / Ключ. ставка`. На `Депозиты`: подсветка средневзв. ставки vs ключевой vs инфляции. Снизу — короткая текстовая заметка с интерпретацией (1–2 предложения), почему именно так. Это единственный модуль, где есть прямой нарративный текст — попытка превратить графики в маленький инсайт.

## Что меняется при переключении вкладок (сводно)

| Tab | Hero показывает | Модули | Что прячется |
|---|---|---|---|
| **Все** | Прирост портфеля | M1 источники (4) + M2 месяцы + M3 лидеры (все 5) + **M4 donut** + M8 bench (4 ориентира) | M5/M6/M7 |
| **Бумаги** | Прирост бумаг | M1 источники (3) + M2 месяцы + M3 лидеры (только sec) + **M7 активность** + M8 bench | M4/M5/M6 + контрибьюторы крипты/прочего |
| **Депозиты** | Накоплено % | M1 источники (1: проценты) + M2 месяцы + **M5 прогресс вкладов** + M8 bench | M3/M4/M6/M7 |
| **Крипта** | Изменение крипты | M1 источники (1: прирост) + M2 месяцы + M3 лидеры (только crypto) + **M6 ATH** + M8 bench | M4/M5/M7 |
| **Прочее** | Прирост | M1 источники (1: прирост) + M2 месяцы + M3 лидеры (только other) + M8 bench | M4/M5/M6/M7 |

Реализовано CSS-правилами на `[data-only="..."]` — JS только переключает `app[data-active]`, остальное делает каскад.

## Интерактив

- **Тема** — переключатель в хедере. `localStorage`: `portfolio-analytics-theme-v1`.
- **Период (segtog в hero)** — обновляет hero number, eyebrow, ascope pill, заголовки sub-карточек, bench note.
- **Asset tabs** — переключают `app[data-active]`. Hero breakdown rows и cont-rows фильтруются CSS-ом, бенчмарк-карточка переписывает title/sub/note. Иконка scope-стрипа получает другой tint.
- **Hero rows** — клик по строке типа активирует соответствующий таб (как в `portfolio-v1`).
- **Monthly bars** — клик на бар обновляет «карточку выбора» под графиком (название месяца, цифра, мета).
- **View toggle** — переключение Позиции / Операции / Аналитика. Этот концепт фокусируется на Аналитике; первые два показывают stub со ссылкой на `portfolio-v1-tinkoff`.

## Что намеренно опущено

- Реальные API и пересчёт. Все цифры замощены под 1Г; смена периода в hero меняет шапку, но не пересчитывает все модули (это разумно для прототипа, в реальности — серверная агрегация).
- Биржевой график цены позиции в bottom-sheet — оставлено детализации позиции, как в `portfolio-v1`.
- Налоговый калькулятор (НДФЛ от продаж, льгота ИИС) — следующий слой, отдельный модуль.
- Сценарный анализ «что если» — не для прототипа.
- Корреляции и риск-метрики (β, sharpe) — для розничного UX это шум.

## Открытые вопросы

1. **Hero морф на view-смене.** Сейчас при переходе Позиции → Аналитика hero полностью перерисовывается (другой eyebrow, другой toggle, другой number). Достаточно ли это очевидно для пользователя или нужен короткий transition (cross-fade, slide)?
2. **Период selector — где жить?** Сейчас он в hero (заменяет «Сейчас/С доходом»). Альтернатива: тонкий стрип над модулями. Преимущество текущего — экономит вертикаль; недостаток — у пользователя нет периодов на других views.
3. **Названия источников дохода.** «Прирост капитала / Купоны / Дивиденды / Проценты по вкладам» — длинно. Сокращать: «Прирост / Купоны / Дивиденды / % по вкладам»? Или «Реализованный / Купоны / Дивиденды / Проценты»?
4. **Monthly bars — направление времени.** Сейчас слева-старее, справа-новее. Это «как у всех», но первое впечатление цепляется за правый край (последний месяц). Может, инвертировать (новое слева)?
5. **Contributors — глубина списка.** Сейчас 3+2. Раскрывать «показать всех» по тапу или достаточно фиксированной выборки?
6. **Бенчмарки по asset-class.** Для бумаг сравниваем с MOEX TR, для депозитов — с ключевой ставкой / инфляцией, для крипты — ?... Сейчас крипта сравнивается с общей картиной просадки от ATH, что концептуально другое. Возможно стоит дать BTC dominance или ETH/BTC ratio как ориентир?
7. **«Дистанция до пика» — только крипта или применимо к акциям тоже?** SBER может быть в просадке от 52-week high — концепт переиспользуем.
8. **Эмпирический объём аналитики.** Сейчас на «Все» — 5 карточек после hero/scope. Это уже скролл на ~3 экрана. Не слишком ли много? Можно вынести часть в «Подробная аналитика →» как отдельный screen?

## Связь с остальной системой

Этот концепт **не** заменяет `portfolio-v1-tinkoff` — он показывает, как должна выглядеть вкладка `Аналитика` в этом портфельном экране. Когда направление утверждено:

1. Перенести модули как React-компоненты в `frontend/src/pages/Portfolio/Analytics/`.
2. Связать с реальными агрегатами на бэкенде (доходность по периодам, источники дохода, monthly P&L).
3. Заменить mock-донат и mbar на ту же визуализацию, но из реальных данных (можно `recharts` или ручное SVG — донат и центр-zero bars выгоднее ручным SVG для контроля стиля).
