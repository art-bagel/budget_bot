# Portfolio — концепт v1 (Tinkoff-leaning)

Редизайн страницы «Портфель» в том же языке, что и `dashboard-v2-tinkoff`: прохладный серый фон, жёлтый акцент `#FFDD2D`, шрифт Onest, плотная информативная подача. Статичный прототип, без сборки.

## Как открыть

Двойной клик по `index.html`. В DevTools — Toggle device toolbar → iPhone 14 Pro / Pixel 7 (375–414 px).

## Что умеет концепт

### 1. Hero: двухрежимный toggle «Сейчас / С доходом»

Вместо старого чипа «30 дней» — сегмент-переключатель. Показывает принципиальную разницу для вкладов и облигаций:

- **Сейчас** — сколько вы получите, если закроете всё прямо сейчас (для депозитов = тело без начисленных процентов, для облигаций = рыночная цена без НКД).
- **С доходом** — сколько будет при закрытии в штатную дату (тело + начисленные проценты для вкладов, цена + НКД для облигаций).

Разница: **+9 260 ₽** (8 420 по 3 вкладам + 840 ₽ НКД по ОФЗ 26238). При переключении обновляются цифры в hero-total, breakdown-строках, группах и позициях; в режиме «С доходом» под hero появляется тонкая плашка с пояснением.

Для акций, крипты и фондов значения в обоих режимах одинаковы — это само по себе полезное обучение: переключатель подсвечивает, что «отложенный доход» бывает только у некоторых инструментов.

### 2. Группировка позиций: по счёту + по подтипу

По запросу — два уровня разреза:

- **Первый уровень (карточка группы)** — по счёту / провайдеру (`Тинькофф Инвестиции`, `Сбер Инвестор`, `Т-Банк`, `Сбербанк`, `Binance`).
- **Второй уровень (подзаголовок внутри группы)** — по подтипу актива для бумаг: `Акции / Облигации / Фонды`. Для депозитов и крипты второй уровень не нужен (каждая позиция самодостаточна).

### 3. Позиция = название + цена штуки × количество

По запросу каждая позиция показывает под названием `цена × количество`:

- Акция: `SBER · 285,00 ₽ × 1 200 шт`
- Облигация: `950,00 ₽ × 100 шт · НКД 8,40 ₽`
- Фонд: `ETF · 1 859,75 ₽ × 80 шт`
- Крипта: `BTC · 6 041 666 ₽ × 0,024`
- Металлы: `7 366 ₽/г × 22,4 г`
- Депозит: ставка + тело + срок (цены за штуку у депозита нет) — `18% · 450 000 ₽ · до 14.07.2026`

### 4. Type summary card — выровненные цифры

По фидбеку: исправлено «разъезжание» сумм на центральном блоке. Теперь `tsum__grid` — жёсткая сетка `1fr 1fr 1fr` с бордерами на средней ячейке вместо псевдо-разделителей. Все 3 значения выровнены по верху, подпись-% (`+7,5%`) живёт под средним значением и не ломает линию.

Карточка показывает:
- Крупно «Сейчас» / «С доходом» (label меняется с режимом) + delta-пилл (30-дневное изменение)
- 3 колонки: `Вложено / Доход / Позиций или Ср.ставка`
- Для депозитов — жёлтая плашка с накопленными % и пояснением, когда они реализуются

### 5. Раздел «Операции» — только инвестиционные

Отдельная карточка под позициями. Показывает последние 5 операций: купон, дивиденды, начисление %, покупка, продажа. Каждая с иконкой-чипом (pos/neg), названием инструмента, датой и счётом. Ссылка «Все →» (в реальном приложении — на полноэкранный список).

Операции фильтруются по активному табу: на `Депозиты` увидите только начисления %, на `Бумаги` — только купоны/дивиденды/сделки и т.д.

### 6. Раздел «Аналитика» — краткий обзор

Ниже операций. Карточка с:
- 4-ячеечная сетка: `Доходность YTD / Пассивный доход / Лучшая позиция / Худшая позиция`
- Сравнительный блок «Портфель vs MOEX» с мини-барами

Ссылка «Подробнее →» (в реальном приложении — на полноэкранный экран аналитики с разрезами по счетам, периодам, типам доходов).

## Интерактив

- **Тема** — переключатель в хедере. `localStorage`: `portfolio-theme-v1`.
- **Segtog «Сейчас ↔ С доходом»** — swap-анимация на hero-total.
- **Tabs** — переключение видимой summary/alloc карточки и фильтрация групп.
- **Hero rows** — клик по строке типа = активировать соответствующий таб.
- **Операции + Аналитика** — ссылки-заглушки для будущей навигации.
- **Клавиатура** — hero-row и tabs фокусируемы, Enter/Space срабатывает.

## Открытые вопросы

1. **Название режима** — «Сейчас / С доходом»? Альтернативы: `Факт / Потенциал`, `Ликвидно / С начислениями`. Какое лучше в UI?
2. **Крипта и фонды в режиме «С доходом»** — пока неотличимы от «Сейчас». Стоит ли учитывать стейкинг-rewards (TON) или накопленные дивиденды фондов, или оставить как сейчас?
3. **Sparkline в summary** — символический (как сейчас) или реальный ряд из истории цен?
4. **Операции — фильтр по табу** — сейчас строгая фильтрация (на `Депозиты` видны только начисления). Может, оставлять все и только подсвечивать активный тип?
5. **Аналитика — виджеты** — правильные ли 4 метрики выбраны (YTD, пассивный доход, лучшая, худшая)? Что ещё добавить: распределение по валютам, среднегодовая доходность, доход по типу (дивиденды/купоны/проценты)?
6. **«Новая» позиция** — куда: bottom-sheet со списком типов или сразу форма?
7. **«Операции»/«Аналитика» ссылки** — открывать отдельный экран (router push) или bottom-sheet-детализацию?
8. **Tinkoff logo на позициях** — подключать `getTinkoffInstrumentLogoUrl` для бумаг или оставлять цветные инициалы как fallback?
