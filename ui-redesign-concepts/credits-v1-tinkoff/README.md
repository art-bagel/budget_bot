# Credits — концепт v1 (Tinkoff-leaning)

Редизайн страницы «Кредиты» в том же языке, что и `dashboard-v2-tinkoff` / `portfolio-v1-tinkoff`: прохладный серый фон, жёлтый акцент `#FFDD2D`, шрифт Onest, плотная информативная подача. Статичный прототип, без сборки.

## Как открыть

Двойной клик по `index.html`. В DevTools — Toggle device toolbar → iPhone 14 Pro / Pixel 7 (375–414 px).

## Главная развилка — два варианта hero

Сверху страницы — dev-only переключатель `Прототип · сравнение hero`. Он переключает hero между двумя направлениями. Выбор сохраняется в `localStorage` (`credits-hero-v1`).

### A · Ink + segtog (по умолчанию)

- Тёмная ink-карточка `linear-gradient(155deg, #15171A → #1F2226)` с тонким жёлтым свечением в правом верхнем углу.
- Сегмент-переключатель **Остаток / С процентами**: левый режим — тело долга на сегодня; правый — тело + начисленные проценты, ответ на «сколько спишут при полном досрочном закрытии прямо сейчас». В режиме «С процентами» появляется тонкая плашка с дельтой.
- Жёлтый прогресс-бар «Погашено тела»: `1 445 000 ₽ из 4 500 000 ₽ · 32%` — только по срочным кредитам, карты возобновляемые (отметка под баром).
- Семантический сигнал: «другая сторона баланса» — без алармизма (не красный), но с ясным контрастом против жёлтого hero на Dashboard / Portfolio.

### B · Жёлтый + прогресс

- Тот же макет, но фон жёлтый, как у Dashboard v2: контекстуальная связь, не выделяет страницу долгом.
- Прогресс-бар инвертирован: ink-заливка на жёлтом.

В обоих вариантах одинаковые: segtog, breakdown по типам, прогресс-бар, плашка-объяснение в режиме «С процентами».

## Структура страницы

1. **Hero** (см. выше).
2. **Tabs + add-button** — `Все · Ипотека · Кредиты · Карты` с жёлтым underline на активной (как `portfolio-v1-tinkoff`). Справа — круглая кнопка `+` для нового кредита.
3. **Секция «Кредиты»** — группы по типу с заголовками (`Ипотека / Кредиты / Карты`). Каждая плитка кликабельна.
4. **Секция «Операции»** — последние 5 операций по кредитам (платежи, начисления процентов, покупки по карте). Фильтруется активным табом. Ссылка `Все →` — заглушка.
5. **Секция «Аналитика»** — 4-cell grid (`Заплачено процентов / Средняя ставка / Платёж в месяц / До конца долга`) + горизонтальный stacked-bar «Тело vs Проценты» по будущим выплатам. Ссылка `Подробнее →`.

## Sheets — все действия

Полностью разделены на отдельные bottom-sheet’ы:

- **Detail sheet** — open по тапу на плитку. Состав: 3-cell grid (Остаток / Проценты / К оплате) → жёлтая плашка следующего платежа (срочные) ИЛИ utilization-bar (карта) → блок «Условия» с кнопкой `Изменить` → action-кнопки `Погасить · Перевод с кредита · График платежей` → archive-action когда долг = 0.
- **Schedule sheet** (full-screen) — открывается из detail. Year-pills сверху (sticky), список платежей с цветной полосой `тело vs проценты` и статусом `Оплачено / Следующий / План`. Принципиально другой layout по сравнению с текущим вложенным modal.
- **Repay sheet** — отдельный sheet с формой погашения. Поля: сумма, валюта, со счёта, дата (только для срочных), комментарий. Внизу sticky-foot с подтверждением.
- **Transfer sheet** — отдельный sheet перевода с карты на счёт. Только для карт. Виден только когда есть свободный лимит.
- **New credit sheet** — segmented-picker `Ипотека · Кредит · Карта` сверху, форма меняется по типу (даты/целевой счёт скрываются для карты).

Backdrop, Esc и кнопки закрытия работают штатно. Sheets могут стекаться (например, открыть Schedule поверх Detail).

## Интерактив

- **Тема** — переключатель в хедере. `localStorage`: `credits-theme-v1`.
- **Hero variant toggle** — `localStorage`: `credits-hero-v1`. Вверху страницы.
- **Segtog «Остаток ↔ С процентами»** — swap-анимация на hero-total и breakdown-строках.
- **Hero rows** — клик по строке типа = активировать соответствующий таб (Ипотека / Кредиты / Карты).
- **Tabs** — фильтрует список кредитов, операций и пересчитывает аналитику (средняя ставка, платёж в месяц, осталось выплатить).
- **Tile** — открывает Detail sheet.
- **Schedule year-pills** — переключают видимый год в графике.
- **Toast** — подтверждение действий (погасить / перевести / создать / архив).
- **Клавиатура** — hero-row, tabs, segtog фокусируемы; Esc закрывает верхний sheet.

## Открытые вопросы

1. **Какой hero оставляем** — A (ink) или B (жёлтый)? Это главная развилка концепта.
2. **Tile · крупно остаток или крупно следующий платёж** — сейчас по запросу долг крупно, платёж в плашке. Если сменить полярность, плитка станет про «что делать сейчас», а не «сколько висит».
3. **Tile-progress по срочным** — сейчас «47 из 240 платежей». Альтернатива: «погашено 23,4% тела» (по сумме, не количеству). Что нагляднее?
4. **Карты в hero / breakdown** — сейчас отдельная строка `Карты 124 500 ₽`. Возможно, их стоит вынести из hero (т.к. они не часть «срочных платежей») и оставить только в списке. Решить.
5. **Аналитика** — правильные ли 4 метрики? Альтернативы: переплата (всего процентов до конца), доля долга в чистом капитале, payment-to-income (если есть зарплата), DTI.
6. **Schedule** — сейчас полный график до конца кредита. Может, ограничить ближайшим годом + кнопкой «показать всё»?
7. **Operations · Покупки по карте** — стоит ли их показывать на странице кредитов, или они ждут блока операций общего? Сейчас они здесь, помечены ink-цветом без знака.
