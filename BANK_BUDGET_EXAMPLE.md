# Bank and Budget Example Flow

Этот файл описывает ключевую идею модели:

- у пользователя есть **общий мультивалютный банк**;
- категории хранят **только бюджет в базовой валюте пользователя**;
- валюты **не привязаны к категориям**;
- при расходе валюта списывается из банка, а категория уменьшается на **историческую себестоимость** списанной валюты в базовой валюте;
- изменение рыночного курса **не меняет категории** и **не создает операций**, а влияет только на отчетную оценку по запросу.

Для общей схемы см. [BANK_BUDGET_MODEL.md](/Users/aleksandrkostenko/Desktop/Dev/budget_bot/BANK_BUDGET_MODEL.md).

## Бизнес-правила

### 1. Банк и бюджет разделены

Есть два независимых слоя учета:

- `bank_entries` — реальные деньги пользователя по валютам;
- `budget_entries` — бюджет категорий в базовой валюте.

Они должны сходиться по **исторической стоимости**, но живут отдельно.

### 2. Категории не хранят валюту

Категории отвечают только на вопрос:

- на какую цель отнести расход;
- сколько бюджетных денег в базовой валюте еще доступно.

Категории не отвечают на вопрос:

- в какой валюте физически лежат деньги.

Это делает общий банк единственным источником истины по валютам.

### 3. Exchange не меняет категории

Обмен валюты:

- меняет только состав денег в банке;
- создает или закрывает валютные лоты;
- не меняет бюджеты категорий.

### 4. Expense списывает две разные сущности

Расход в валюте делает сразу две вещи:

- уменьшает банк в валюте расхода;
- уменьшает выбранную категорию в базовой валюте на историческую себестоимость списанной валюты.

### 5. Рыночный курс не участвует в учете категорий

Если курс изменился:

- `bank_entries` не меняются;
- `budget_entries` не меняются;
- `fx_lots` не меняются.

Меняется только ответ в отчете:

- сколько все текущие деньги стоят сейчас в `RUB`, `USD`, `CNY` и т.д.

### 6. Нереализованный FX-результат не попадает в категории

Пока валюта не продана обратно:

- прибыль или убыток по курсу считается только аналитически;
- бюджет категорий остается на исторической стоимости.

Реализованный FX-результат появляется только в момент обратного обмена.

## Пример

Предположим:

- базовая валюта пользователя: `RUB`
- есть основной счет `Main`
- есть системная категория `Unallocated`
- есть группа `Бытовые расходы`
- в группе состоят:
  - `Продукты` на `30000 RUB`
  - `Жилье` на `40000 RUB`
  - `Транспорт` на `10000 RUB`

### Состав сущностей

Пусть идентификаторы такие:

- `user_id = 1`
- `bank_account_id = 1`
- `category_id = 10` -> `Unallocated`
- `category_id = 11` -> `Продукты`
- `category_id = 12` -> `Транспорт`
- `category_id = 13` -> `Жилье`
- `category_id = 14` -> `Бытовые расходы`

### Доли группы

В `group_members` заранее настроено:

- `14 -> 11`, `share = 0.375`
- `14 -> 13`, `share = 0.500`
- `14 -> 12`, `share = 0.125`

Это дает распределение `80000 RUB` как:

- `30000 RUB` в `Продукты`
- `40000 RUB` в `Жилье`
- `10000 RUB` в `Транспорт`

## Шаг 1. Получили 100000 RUB на счет

Создается операция `income`.

### Что пишется в БД

`operations`

```text
id=100
user_id=1
type='income'
comment='Зарплата'
```

`bank_entries`

```text
(operation_id=100, bank_account_id=1, currency_code='RUB', amount=100000)
```

`budget_entries`

```text
(operation_id=100, category_id=10, currency_code='RUB', amount=100000)
```

### Что это значит

- реальные деньги пришли в банк;
- бюджет пока не распределен по целевым категориям, поэтому идет в `Unallocated`.

### Состояние

Банк:

- `100000 RUB`

Бюджет:

- `Unallocated = 100000`
- `Продукты = 0`
- `Жилье = 0`
- `Транспорт = 0`

## Шаг 2. Обменяли 80000 RUB на 1000 USD

Курс рассчитывается автоматически:

```text
80000 / 1000 = 80 RUB за 1 USD
```

Создается операция `exchange`.

### Что пишется в БД

`operations`

```text
id=101
user_id=1
type='exchange'
comment='Обмен RUB -> USD'
```

`bank_entries`

```text
(operation_id=101, bank_account_id=1, currency_code='RUB', amount=-80000)
(operation_id=101, bank_account_id=1, currency_code='USD', amount=1000)
```

`fx_lots`

```text
id=500
bank_account_id=1
currency_code='USD'
amount_initial=1000
amount_remaining=1000
buy_rate_in_base=80
cost_base_initial=80000
cost_base_remaining=80000
opened_by_operation_id=101
```

### Что это значит

- в банке стало меньше рублей и появились доллары;
- бюджет категорий не меняется;
- создан валютный лот с исторической себестоимостью.

### Состояние

Банк:

- `20000 RUB`
- `1000 USD`

Историческая стоимость банка:

- `20000 RUB + 80000 RUB = 100000 RUB`

Бюджет:

- `Unallocated = 100000`
- `Продукты = 0`
- `Жилье = 0`
- `Транспорт = 0`

## Шаг 3. Закинули 80000 RUB на группу

Создается операция `group_allocate`.

### Что пишется в БД

`operations`

```text
id=102
user_id=1
type='group_allocate'
comment='Распределение по группе Бытовые расходы'
```

`budget_entries`

```text
(operation_id=102, category_id=10, currency_code='RUB', amount=-80000)
(operation_id=102, category_id=11, currency_code='RUB', amount=30000)
(operation_id=102, category_id=13, currency_code='RUB', amount=40000)
(operation_id=102, category_id=12, currency_code='RUB', amount=10000)
```

### Что это значит

- деньги в банке не двигаются;
- меняется только бюджетное распределение.

### Состояние

Банк:

- `20000 RUB`
- `1000 USD`

Бюджет:

- `Unallocated = 20000`
- `Продукты = 30000`
- `Жилье = 40000`
- `Транспорт = 10000`

## Шаг 4. Потратили с Продуктов 10000 RUB

Создается операция `expense`.

### Проверки

- в банке есть `10000 RUB`;
- в категории `Продукты` есть минимум `10000 RUB` бюджета.

### Что пишется в БД

`operations`

```text
id=103
user_id=1
type='expense'
comment='Покупка продуктов'
```

`bank_entries`

```text
(operation_id=103, bank_account_id=1, currency_code='RUB', amount=-10000)
```

`budget_entries`

```text
(operation_id=103, category_id=11, currency_code='RUB', amount=-10000)
```

### Что это значит

- банк уменьшается на реальные рубли;
- категория `Продукты` уменьшается на ту же сумму.

### Состояние

Банк:

- `10000 RUB`
- `1000 USD`

Бюджет:

- `Unallocated = 20000`
- `Продукты = 20000`
- `Жилье = 40000`
- `Транспорт = 10000`

## Шаг 5. Потратили с Жилья 200 USD

Создается операция `expense`.

### Проверки

- в банке есть `200 USD`;
- в категории `Жилье` хватает бюджета на историческую себестоимость списания.

Историческая себестоимость:

```text
200 USD * 80 RUB = 16000 RUB
```

Категория `Жилье` имеет `40000 RUB`, значит операция разрешена.

### Что пишется в БД

`operations`

```text
id=104
user_id=1
type='expense'
comment='Расход по жилью в USD'
```

`bank_entries`

```text
(operation_id=104, bank_account_id=1, currency_code='USD', amount=-200)
```

`lot_consumptions`

```text
(operation_id=104, lot_id=500, amount=200, cost_base=16000)
```

`fx_lots` обновляется:

```text
id=500
amount_remaining=800
cost_base_remaining=64000
```

`budget_entries`

```text
(operation_id=104, category_id=13, currency_code='RUB', amount=-16000)
```

### Что это значит

- физически из банка ушли `200 USD`;
- экономически расход отнесен на категорию `Жилье` в размере `16000 RUB`;
- рыночный курс в момент расхода здесь не используется.

### Состояние

Банк:

- `10000 RUB`
- `800 USD`

Лоты:

- `lot 500 = 800 USD`
- `cost_base_remaining = 64000 RUB`

Бюджет:

- `Unallocated = 20000`
- `Продукты = 20000`
- `Жилье = 24000`
- `Транспорт = 10000`

## Итог после всех шагов

Фактические деньги:

- `10000 RUB`
- `800 USD`

Историческая стоимость банка:

```text
10000 RUB + 64000 RUB = 74000 RUB
```

Итоговый бюджет:

- `Unallocated = 20000`
- `Продукты = 20000`
- `Жилье = 24000`
- `Транспорт = 10000`

Сумма бюджета:

```text
20000 + 20000 + 24000 + 10000 = 74000 RUB
```

Банк и бюджет сошлись по исторической стоимости.

## Что будет при изменении курса

Если курс вырос, например, до `95 RUB/USD`, это **не создает новых операций**.

Учет остается тем же:

- банк: `10000 RUB + 800 USD`
- бюджет: `74000 RUB`
- историческая стоимость банка: `74000 RUB`

Но в отчете по текущему рынку можно показать:

```text
10000 + 800 * 95 = 86000 RUB
```

Это только аналитическая оценка, а не изменение бюджета.

## Ключевой вывод

В этой модели:

- банк отвечает за реальные деньги и валюты;
- категории отвечают за бюджет в базовой валюте;
- расход в валюте всегда списывает валюту из банка;
- категория уменьшается на историческую себестоимость этой валюты;
- рыночный курс нужен только для аналитики и отчетов.
